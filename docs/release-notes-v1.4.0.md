# v1.4.0 发布说明 — 禁用 / 恢复 IDM 更新提示

- 发布日期：2026-07-24
- 类型：运行时功能新增（需要重新下载发布包）
- 对应反馈：[issue #20](https://github.com/tytsxai/IDM-Activation-Script-Chinese/issues/20)「更新太频繁 总是弹窗更新，能够禁用掉就好了」

## 背景

IDM 默认会定期联网检查新版本并弹出更新提示。对使用本脚本的用户而言这有两层困扰：

1. 弹窗本身频繁、打断使用；
2. 一旦真的升级到新版本，脚本此前写入的激活/冻结状态经常失效，需要重新跑一遍。

在 v1.4.0 之前，本仓库没有提供任何关闭更新检查的入口，用户只能自己去改注册表。

## 本次改动

### 新增功能

`IAS.cmd` 主菜单新增两项：

```
[4] 禁用 IDM 更新提示
[5] 恢复 IDM 更新提示
```

命令行等价参数：`IAS.cmd /noupd`（禁用）、`IAS.cmd /reupd`（恢复），可与 `/silent` `/log` 组合无人值守执行。同时给出两个参数时以 `/noupd` 为准。

### 实现方式

把注册表值 `HKCU\Software\DownloadManager\CheckUpdtVM` 置为 `0`（恢复时置 `1`）。这是 IDM 自己的"自动检查更新"开关，置 0 后 IDM 不再联网检查新版本，也就不会弹出更新提示窗。

具体行为：

- 先检查 IDM 是否已安装（未安装则报错退出，退出码 `1`）；
- 读取并打印/记录变更前的原值，便于事后追溯；
- 结束正在运行的 `IDMan.exe`，让设置在 IDM 下次启动时生效；
- 当 `HKCU` 与 `HKU\<SID>` 不同步时（脚本启动阶段已探测），两处都写入；
- 每次写入结果都打印成功/失败并写入日志，失败时置退出码 `1`。

### 边界

- **不写序列号、不动 CLSID 注册表键**，与激活 / 冻结 / 重置流程完全独立，不会影响已有的激活状态；
- **不修改任何 IDM 程序文件**，也不改 hosts、防火墙或系统代理；
- 关掉更新检查后不再获得官方修复与新功能，用 `[5]` 可随时改回。

### 菜单编号调整

`[4]` `[5]` 让位给更新开关，原有条目顺延：

| 功能 | v1.3.9 | v1.4.0 |
| --- | --- | --- |
| 激活（冻结） | `[1]` | `[1]` |
| 激活 | `[2]` | `[2]` |
| 重置激活/试用期 | `[3]` | `[3]` |
| 禁用 IDM 更新提示 | — | `[4]` |
| 恢复 IDM 更新提示 | — | `[5]` |
| 下载 IDM | `[4]` | `[6]` |
| 帮助 | `[5]` | `[7]` |
| 退出 | `[0]` | `[0]` |

菜单窗口高度由 28 行调整为 31 行。激活/冻结成功后的提示追加一行，提醒可用 `[4]` 关闭更新以避免升级后失效。

## 回归建议（Windows 真机）

1. 菜单能正常显示 8 个选项，按 `1`–`7`、`0` 都能正确分派；
2. 选 `[4]` 后：`reg query "HKCU\Software\DownloadManager" /v CheckUpdtVM` 应为 `0x0`；重开 IDM 不再弹更新提示；
3. 选 `[5]` 后该值应为 `0x1`；
4. 先 `[2]` 激活再 `[4]` 禁用更新，IDM 激活状态应保持不变；
5. 未安装 IDM 时选 `[4]` 应提示"IDM 未安装"并返回码 `1`；
6. `IAS.cmd /noupd /silent /log="C:\Temp\ias.log"` 能无人值守完成，日志里有变更前原值与写入结果。

## 升级建议

v1.4.0 是运行时改动。已在使用旧版的用户请重新下载发布包（附 `.sha256`）。已激活的用户重新下载后无需重新激活，只需在菜单里选 `[4]` 关闭更新提示即可。
